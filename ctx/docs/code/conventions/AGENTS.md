# Code Conventions (`./ctx/docs/code/conventions/`)

Path: `./ctx/docs/code/conventions/AGENTS.md`

## Purpose

The `ctx/docs/code/conventions/` directory contains declarative conventions that govern recurring implementation forms used across the codebase. Documents at this level define stable coding patterns and static-analysis rules that must remain consistent across modules and packages.

## Level Map

- `AGENTS.md` — this document, defining the boundaries and structure of the `conventions/` level.
- `es6-modules.md` — normative structural form for DI-compatible ES modules implemented in JavaScript.
- `types-map.md` — convention for mapping architectural namespace identifiers to static type declarations.

## Level Boundary

This level defines reusable code conventions only. It must not describe repository topology, package-specific metadata, runtime environment, product semantics, or architectural contours.

## Summary

`ctx/docs/code/conventions/AGENTS.md` defines the scope and navigation of cross-cutting implementation conventions.
