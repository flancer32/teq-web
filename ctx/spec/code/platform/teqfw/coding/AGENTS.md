# TeqFW Coding Specifications

- Path: `ctx/spec/code/platform/teqfw/coding/AGENTS.md`

## Purpose

This level defines code-form conventions required for TeqFW-compatible implementation modules and package metadata.

## Level Map

- `AGENTS.md` — level definition for TeqFW coding specifications.
- `component-types.md` — architectural component categories and their publication model.
- `es6-module.md` — canonical ES module structure used as a TeqFW publication unit.
- `jsdoc.md` — JSDoc conventions for structural typing in runtime JavaScript.
- `package-json.md` — Node.js package manifest conventions for TeqFW packages.

This level defines code representation and publication forms.

This level does NOT define addressing semantics or runtime assembly behavior. Those concerns belong to `addressing/` and `composition/`.
